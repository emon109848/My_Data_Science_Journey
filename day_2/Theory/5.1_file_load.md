## 1Ô∏è‚É£ Saving NumPy Arrays

### 1Ô∏è‚É£ Binary Format (`.npy`) ‚Äî Fast and Efficient

```python
import numpy as np

arr = np.array([1,2,3,4,5])
np.save("array.npy", arr)   # saves in binary format
```

* **Pros:** Fast, preserves dtype, efficient for large arrays
* **Load back:**

```python
loaded_arr = np.load("array.npy")
```

---

### 2Ô∏è‚É£ Text Format (`.txt` or `.csv`) ‚Äî Human-readable

```python
np.savetxt("array.txt", arr, delimiter=",")  # CSV format
```

* **Pros:** Human-readable, easy to share
* **Cons:** Slower, dtype may convert to float
* **Load back:**

```python
loaded_arr = np.loadtxt("array.txt", delimiter=",")
```

---

## 2Ô∏è‚É£ CSV Handling in NumPy

* Use **`savetxt()`** for CSV export
* Use **`loadtxt()`** for CSV import
* Example:

```python
data = np.array([[1,2,3],[4,5,6]])
np.savetxt("data.csv", data, delimiter=",")
loaded_data = np.loadtxt("data.csv", delimiter=",")
```

* Works for **numeric data only**
* For mixed types ‚Üí use **pandas**

---

## 3Ô∏è‚É£ Text vs Binary Format

| Feature         | `.npy` (Binary)          | `.txt` / `.csv` (Text)      |
| --------------- | ------------------------ | --------------------------- |
| Speed           | Fast                     | Slower                      |
| File Size       | Smaller                  | Larger                      |
| Human-readable  | No                       | Yes                         |
| Data Integrity  | Preserves dtype exactly  | May convert to float        |
| Recommended for | Large arrays / ML models | Small arrays / sharing data |

## 5Ô∏è‚É£ Quick Tips

* Use `.npy` for **NumPy-to-NumPy storage**
* Use `.txt`/`.csv` for **interoperability**
* For multiple arrays ‚Üí use `np.savez()` / `np.savez_compressed()`

```python
np.savez("arrays.npz", arr1=arr, arr2=data)
loaded = np.load("arrays.npz")
print(loaded["arr1"])
```
