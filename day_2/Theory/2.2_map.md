# ğŸ§  What is `map()`?

* **Functional programming tool**
* Applies a **function to each item** in an iterable
* Returns a **map object** (lazy iterator)

* `function` â†’ a callable (normal or lambda function)
* `iterable` â†’ list, tuple, set, string, etc.

# ğŸ”¹ 5ï¸âƒ£ Lazy Evaluation

* `map()` **does not compute immediately**
* Generates items on-demand
* Useful for **large datasets**

```python

# computes values only when iterated
```

# ğŸ”¹ 7ï¸âƒ£ Map vs List Comprehension

| Map                       | List Comprehension   |
| ------------------------- | -------------------- |
| Functional style          | Pythonic style       |
| Can use existing function | Inline logic         |
| Lazy evaluation           | Immediate evaluation |
| Often paired with lambda  | Can be more readable |

Example equivalent:

```python
list(map(lambda x: x**2, nums))
```

```python
[x**2 for x in nums]
```

# ğŸ§  INTERNAL WORKING

1. `map()` stores reference to function & iterable
2. Returns **map object**
3. Computes values **only on iteration**

```python

next(m)  # 1
next(m)  # 4
```
